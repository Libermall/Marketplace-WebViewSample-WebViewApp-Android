# Диагностический отчёт проекта LibermallNFT WebView App

## Обзор проекта

Данный репозиторий содержит несколько Android WebView приложений для маркетплейса LibermallNFT:

1. **WebViewSample** - основное приложение (`com.alllibermall.libermalplaceall`)
2. **com.libmallnftscshop.nftlibermall** - альтернативная версия приложения
3. Дополнительные папки `phone/` и `tablet/` для разных устройств

## Архитектура

- **Тип приложения**: Android WebView wrapper для веб-сайта libermall.com
- **Язык программирования**: Kotlin
- **Минимальная версия Android**: API 21 (Android 5.0)
- **Целевая версия Android**: API 32-33
- **Основные компоненты**:
  - MainActivity - основная активность с WebView
  - MyFirebaseMessagingService - сервис push-уведомлений
  - Поддержка загрузки файлов и мультимедиа

## 🚨 КРИТИЧЕСКИЕ ПРОБЛЕМЫ БЕЗОПАСНОСТИ

### 1. Хранение паролей в открытом виде
**Уровень риска: КРИТИЧЕСКИЙ**
- Файлы `password.txt` содержат пароли в открытом виде: "password 12345678"
- Пароли доступны любому, кто имеет доступ к коду
- **Рекомендация**: Немедленно удалить все пароли из репозитория

### 2. Небезопасные настройки WebView
**Уровень риска: ВЫСОКИЙ**

```kotlin
settings.allowFileAccessFromFileURLs = true
settings.allowUniversalAccessFromFileURLs = true
```

- Эти настройки открывают возможности для атак типа Cross-Site Scripting (XSS)
- Позволяют веб-странице получать доступ к локальным файлам

### 3. Автоматическое предоставление разрешений
**Уровень риска: ВЫСОКИЙ**

```kotlin
override fun onPermissionRequest(request: PermissionRequest) {
    request.grant(request.resources)
}
```

- Автоматически предоставляет ВСЕ запрашиваемые разрешения без проверки
- Может привести к несанкционированному доступу к камере, микрофону, местоположению

### 4. Разрешение небезопасного трафика
**Уровень риска: СРЕДНИЙ**

```xml
android:usesCleartextTraffic="true"
```

- Разрешает передачу данных по незашифрованным каналам
- Делает приложение уязвимым для атак "человек посередине"

### 5. Хранение ключей подписи в репозитории
**Уровень риска: КРИТИЧЕСКИЙ**
- Файлы `key.jks` находятся в открытом репозитории
- Это позволяет любому создавать поддельные версии приложения

### Безопасность WebView:
```kotlin
// Безопасные настройки WebView
settings.allowFileAccessFromFileURLs = false
settings.allowUniversalAccessFromFileURLs = false
settings.allowFileAccess = false // если не требуется доступ к файлам

// Проверка разрешений
override fun onPermissionRequest(request: PermissionRequest) {
    // Проверять каждое разрешение индивидуально
    val allowedResources = request.resources.filter { resource ->
        when (resource) {
            PermissionRequest.RESOURCE_AUDIO_CAPTURE,
            PermissionRequest.RESOURCE_VIDEO_CAPTURE -> true
            else -> false
        }
    }
    if (allowedResources.isNotEmpty()) {
        request.grant(allowedResources.toTypedArray())
    } else {
        request.deny()
    }
}
```
